# Threat Modeler

This skill enables Gemini CLI to perform structured threat modeling on system architecture diagrams generated by the "Board" application.

## Workflow

1.  **Ingestion**: Read the Board JSON file (e.g., `result.json`).
2.  **Analysis**: Analyze the nodes and edges using the STRIDE methodology.
    -   Reference [references/stride.md](references/stride.md) for framework details.
    -   Reference [references/schema.md](references/schema.md) for JSON structure details.
3.  **Modeling**:
    -   Identify **Threats** (e.g., `SQL Injection`, `Credential Leakage`). Output them for review.
4.  **Integration**:
    -   Generate new nodes for identified security threats.
    -   Attach them to appropriate parents using `parentId`.
    -   Place them at `position: { x: 10, y: 10 }` (they will be offset by the application).
5. **Output**: Save the enriched JSON back to a file (e.g., `enriched_result.json`).
6.  **Verification**:
    -   Run the validation script to ensure the JSON is valid and formatted correctly.
    ```bash
    node threat-modeler/scripts/validate.cjs <path-to-json>
    ```

## Example Output Node

```json
{
  "id": "threat_unique_id",
  "type": "threat",
  "position": { "x": 10, "y": 10 },
  "data": {
    "label": "Threat Name",
    "description": "Details about the threat."
  },
  "style": { "zIndex": 10 },
  "parentId": "target_node_id",
  "extent": "parent"
}
```
